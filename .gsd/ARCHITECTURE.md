## Data Flow

1. **Input:** User provides load profiles (`dadosnovos.csv` or `.mat`) via the UI configuration.
2. **Pre-Processing:** The `SimulationManager` routes the data to the `BessEMS` to apply operation strategies (Load Shifting, Peak Shaving).
3. **Execution:** The calculated power load is sent month-by-month to the specified backend (PLECS via XML-RPC or the built-in python RC-circuit equivalent).
4. **Degradation:** High-resolution SOC curves outputted by the backend are processed by `rainflow` algorithms to extract microcycles.
5. **Accumulation:** The `degradation_model` computes severity and incremental cyclic/calendar damage based on physics laws as restricted by the application.
6. **Output:** The result is relayed back to `streamlit_app.py` incrementally and stored redundantly in `results/<sim_id>`.
